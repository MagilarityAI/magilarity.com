# 🪄 MAGILARITY
## Магічна прозорість державних закупівель через штучний інтелект

> *Magilarity = Magic + Clarity — магічна прозорість там де її навмисно приховують*

**MAGILARITY** — це виробнича мультиагентна AI-платформа для автоматизованого юридичного аналізу системи публічних закупівель України. Система аналізує тендери ProZorro, рішення регуляторних органів, відповідність законодавству та документи учасників за допомогою спеціалізованих AI-агентів і оркестрації великих мовних моделей.

---

## 🎯 Проблема яку ми вирішуємо

Система публічних закупівель України (ProZorro) щорічно обробляє сотні тисяч тендерів на суму понад **800 мільярдів гривень**. Виявлення порушень — змови учасників, дискримінаційних умов тендерної документації, заточки специфікацій під конкретного постачальника, антиконкурентних узгоджених дій — вимагає глибокої юридичної експертизи, аналізу сотень сторінок документів і перехресних посилань на регуляторні рішення та законодавство.

**Зараз ця робота займає у досвідченого юриста 8–16 годин на один тендер.**

### Системні проблеми ринку:

- 🔴 **Приховані дискримінаційні вимоги** — замовники свідомо ховають умови що обмежують конкуренцію в різних розділах тендерної документації
- 🔴 **Заточка специфікацій** — технічні характеристики описуються так щоб підходити лише одному постачальнику
- 🔴 **Антиконкурентні змови** — учасники координують ціни та стратегії подання пропозицій
- 🔴 **Односторонні договори** — проекти договорів містять умови що захищають виключно інтереси замовника
- 🔴 **Недоступність юридичної допомоги** — малий та середній бізнес не може дозволити собі профільного юриста для кожного тендеру

**MAGILARITY автоматизує весь цей процес.**

## 🔬 TenderDoc Researcher — як працює

Аналізує тендерну документацію по **13 категоріях закупівель** (будівельні роботи, продукти харчування, фармацевтика, IT послуги, паливо, медичне обладнання тощо).

### 11-кроковий pipeline:

1. **Класифікація** — визначення коду ДК та категорії закупівлі
2. **Перевірка реєстру** — дедублікація
3. **Завантаження** — повний JSON ProZorro + всі документи тендеру
4. **Витяг тексту** — python-docx (основний), pdfplumber, openpyxl
5. **Отримання прецедентів ООПЗ** — релевантні рішення з бази даних
6. **Профілювання замовника** — аналіз історії закупівель
7. **Аналіз Q&A** — обробка роз'яснень замовника та змін до ТД
8. **Основний аналіз (2 виклики LLM)** — 16-точковий юридичний аналіз + перелік документів
9. **Аналіз договору** — юридичний огляд проекту договору
10. **Генерація звіту** — 6 вихідних файлів
11. **Оновлення реєстру** + передача до bid_researcher

### 16 критеріїв аналізу включають:

- Порушення законодавства (Закон №922-VIII, Постанова КМУ №1178)
- Обмеження конкуренції та дискримінаційні вимоги
- Індикатори заточки специфікації (під конкретного постачальника)
- Надмірні кваліфікаційні вимоги
- Справедливість критеріїв оцінки (питома вага ціни ≥70%)
- Вимоги до локалізації (Постанова КМУ №782)
- Підстави для оскарження з дедлайнами (3 дні до кінця подання за воєнного стану)

### 🌟 Унікальний модуль Equipment Investigator

Для специфічного обладнання агент автоматично:
- Витягує технічні характеристики з ТД
- Ідентифікує марку та модель обладнання через веб-пошук
- Знаходить офіційних дистриб'юторів в Україні
- Виявляє ознаки заточки специфікації під конкретного виробника
- Генерує готовий текст для скарги до ООПЗ

### Підтримка донорських програм

Окремий модуль для закупівель що фінансуються міжнародними донорами:
- 🇺🇦 Ukraine Facility (ЄС)
- 🏦 ЄБРР, Світовий банк, ЄІБ
- 🇺🇸 USAID
- 🇺🇳 ООН, KfW

**Вихідні файли:** `analysis_report.docx`, `documents_checklist.docx`, `contract_analysis.docx`, `winner_checklist.docx`, `chronological_checklist.docx`, `analysis.json`

## 📋 Bid Researcher — як працює

Отримує перелік документів від tender_doc_researcher і аналізує подану пропозицію кожного учасника.

### Каскадний pipeline (архітектура сит):

```
Етап 1: Індексація файлів    → 1 виклик на файл → doc_index.json
Сито 1: Прості документи     → 1 текстовий виклик → підтверджено / в обробці
Сито 2: Табличні документи   → пакети по 8 PDF → підтверджено / в обробці
Сито 3: Складні скани        → пакети по 8 PDF → підтверджено / не вирішено
Арбітр                       → послідовно для кожного невирішеного → фінальний вердикт
Фінальний звіт               → JSON + DOCX
```

### Ключові можливості:

- Виявлення інформації у складових документах (правова основа: ст.16 Закону №922-VIII)
- Порівняння технічних специфікацій через vision LLM (PDF скани → зображення)
- Верифікація вмісту документів: банківські гарантії, кваліфікаційні критерії, форми
- Сканування ЕЦП (.p7s файли) з успадкуванням для вмісту архівів
- Інкрементальний аналіз: пропозиція → корекція 24год → документи переможця
- **Механізм SAFEGUARD** — захист від помилок LLM при аналізі банківських гарантій

**Шкала вердиктів:** `відповідає` → `потребує_корекції` → `не_відповідає` → `потребує_перевірки_людиною`

**Вихід на учасника:** `{єдрпоу}_analysis.json` (15+ розділів), `{єдрпоу}_report.docx` (15+ глав, A4 альбомна)

## ⚖️ OOPZ Researcher — як працює

Обробляє рішення Органу оскарження публічних закупівель (ООПЗ) і формує базу даних регуляторних прецедентів яку використовують інші агенти.

### 10-кроковий аналіз кожного рішення:

1. Аналіз LLM зі строгими правилами проти галюцинацій
2. Юридична валідація — всі посилання на норми перевіряються по базі законодавства
3. Гібридна оцінка якості (70% LLM + 30% незалежний валідатор)

**Система оцінок:** A+ (9-10), A (8-8.9), B+ (7-7.9)...

**Вихід:** детальний звіт DOCX, JSON для споживання агентами, реєстр Excel, зберігання PostgreSQL
