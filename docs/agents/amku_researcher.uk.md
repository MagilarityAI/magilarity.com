# ⚔️ AMKU Researcher
**Статус:** В розробці (ядро пайплайну працює, чекає live-валідації + масового наповнення)
**Роль:** Формує базу знань доказів і логіки доказування для агента `investigation` з рішень АМКУ про змови на торгах

---

## Призначення

Антимонопольний комітет України (АМКУ) видає рішення щодо антиконкурентних узгоджених дій у публічних закупівлях (змови на торгах). `amku_researcher` опрацьовує ці рішення і витягує не лише факти, а й **логіку доказування**, якою користуються спеціалісти АМКУ — включно з запереченнями відповідачів та тим, як АМКУ їх спростовує.

**Скоуп свідомо вузький — це рішення, а не обмеження:** лише п.4 ч.2 ст.6 Закону України «Про захист економічної конкуренції» (антиконкурентні узгоджені дії / змови на торгах). Агент має поглиблюватись саме в цей тип порушення, а не розширюватись на інші справи АМКУ.

`amku_researcher` не оцінює рішення — висновки АМКУ вже остаточні. Його завдання — видобути патерни доказів та логіку доведення, щоб `investigation` міг застосувати ту саму логіку при побудові нових справ.

```
        ↓ (зчитується)

```

## Архітектура (потік)

```
main.py (регекс-витяг метаданих: номер рішення, дата, сторони)
        ↓
analyze_decision.py
    → LLM: Gemini 2.5 Flash, ЦІЛЕ рішення (один виклик, без chunking)
    → фолбек: OpenAI, через LLM_PROVIDER
        ↓
validate.py (перевірка структури відповіді LLM)
        ↓
результат JSON v2
   ├── agents_docs/amku_research_results/ (локальні JSON/DOCX/Excel звіти)
   └── amku_memory_store.py → agent_memory.amku_bid_rigging_knowledge (agent_id='amku_researcher')
```

**Full-text аналіз (відремонтовано 07.07.2026):** попередня архітектура різала рішення на частини (chunking) перед аналізом, що втрачало перехресні зв'язки між доказами та висновками. Поточний пайплайн передає весь текст рішення в Gemini 2.5 Flash одним викликом.

## Що фіксує таблиця знань (промпт v2, 07.07.2026)

Кожне проаналізоване рішення дає:

| Елемент | Опис |
|---------|------|
| Докази (`докази`) | Прямі цитати, класифіковані за силою |
| Логічні ланцюги | Послідовності аргументів з нормами права на кожній ланці |
| Заперечення відповідачів | Що стверджували звинувачені сторони у свій захист |
| Спростування АМКУ | Як комітет розбирав кожне заперечення |
| Штрафи з обґрунтуванням | Розміри санкцій та мотивація АМКУ (виключно в межах п.4 ч.2 ст.6) |

Ця комбінація — доказ + контраргумент + спростування — робить таблицю корисною як навчальний матеріал для `investigation`, якому доведеться передбачати і випереджати ті самі заперечення при побудові власних справ.

## 🛡️ Правила проти галюцинацій

- Кожне доказове твердження має підтверджуватись прямою цитатою з тексту рішення.
- Норми права витягуються **як зазначено в рішенні** — «фотографія» логіки АМКУ. Незалежна юридична валідація НЕ виконується (свідоме рішення: рішення АМКУ вже остаточні; валідація належить агентам, які застосовують норми заново, насамперед `investigation`).
- Дата рішення не знайдена → порожньо + попередження. Ніколи не підставляється сьогоднішня дата.
