## 🔄 В розробці — Завершення Фази 1

### amku_researcher (90% готовності)
Дослідження та аналіз рішень Антимонопольного комітету України щодо антиконкурентних узгоджених дій у сфері публічних закупівель.

---

### td_generator — Генератор тендерної документації
Генерує відповідну тендерну документацію використовуючи базу вимог накопичену з аналізів tender_doc_researcher. База фіксує які вимоги використовуються в яких категоріях закупівель, створюючи систему шаблонів на основі даних.

**Джерело даних:** таблиця `prozorro_intel.td_requirements` — наповнюється автоматично під час кожного прогону tender_doc_researcher:
- Текст вимоги
- Категорія ДК
- Кількість використань по тендерах
- Чи є вимога законно допустимою

---

### investigation — Оркестратор розслідування змов
Повний 10-етапний pipeline для виявлення порушень у закупівлях:

| Етап | Аналіз |
|------|--------|
| 1-3 | Ідентифікація тендеру, аналіз учасників |
| 4 | Точка рішення: виявлено індикатори порушень? |
| 5 | Глибокий аналіз: кореляція IP, тайминг подачі, метадані |
| 6 | Фінансові зв'язки, послідовності банківських гарантій |
| 7 | Технічний аналіз документів: ідентичні помилки, спільні шаблони |
| 8 | Фінансовий аналіз: цінові патерни, фінансові зв'язки |
| 9 | Точка рішення: достатньо доказів? |
| 10 | Синтез доказів → пакет скарги до АМКУ |

**9 категорій порушень:**
- Цінова змова (координація цін, ротація переможців)
- Технічна координація (ідентичні специфікації, спільні помилки)
- Поділ ринку (географічний/категорійний розподіл)
- Дискримінація учасників (незаконні бар'єри)
- Конфлікт інтересів (аффілійованість з замовником)
- Удавана конкуренція (підставні учасники)
- Маніпуляція документами (фальсифікація)
- Порушення строків
- Інші процедурні порушення

**4 правові стратегії:** Агресивна (30-50% успіху), Помірна (50-70%), Консервативна (70-85%), Мирова (85-95%)

---

### bid_doc_preparer — Підготовка пакету документів учасника
Готує повні пакети документів для учасників тендеру на основі:
- Чекліста від tender_doc_researcher
- Шаблонів документів
- Наявного архіву документів учасника

---

### Інтеграція YouControl API
Інтеграція з YouControl — платформою корпоративної інформації України — для автоматизованої верифікації учасників тендерів і замовників:

| Тип даних | Призначення |
|-----------|-------------|
| Реєстраційні дані компанії | Верифікація ЄДРПОУ, правового статусу, дати реєстрації |
| Бенефіційні власники | Виявлення конфліктів інтересів, афілійованих компаній |
| Судові рішення | Ідентифікація судової історії, боргових спорів |
| Фінансова звітність | Верифікація заяв про фінансову спроможність |
| Статус ліцензій і дозволів | Перевірка кваліфікаційних вимог |
| Санкційний скринінг | Верифікація відсутності російських санкцій |

**Використовується:** агентом `investigation` (скринінг учасників), `bid_researcher` (профілювання замовника), `tender_doc_researcher` (модуль customer_profiler)

## 🚀 Фаза 2 — Масштабування на ринку України

### Розширення функціональності платформи

**Повний customer_profiler — Профіль замовника**
- Рейтинг репутації замовника на основі рішень ООПЗ, судових справ і статистики закупівель
- Виявлення патернів зловживань конкретного замовника
- Прогнозування поведінки замовника для нових тендерів
- Категоризація: trusted / caution / high_risk

**Агент розгляду скарг ООПЗ**
- AI-помічник для членів колегії ООПЗ при розгляді скарг
- Автоматичний пошук аналогічних прецедентів
- Проект резолютивної та мотивувальної частини рішення
- Забезпечення послідовності практики ООПЗ

**Автоматична подача скарг до ООПЗ**
- Генерація повного пакету скарги на основі аналізу tender_doc_researcher
- Автоматичне заповнення форм
- Відстеження статусу розгляду

**Веб-інтерфейс та кабінет користувача**
- Streamlit → повноцінний React фронтенд
- Особистий кабінет з історією аналізів
- Панель аналітики для юридичних фірм
- API для інтеграцій третіх сторін

---

### Розширення баз даних

**База судових рішень**
- Рішення адміністративних судів де стороною є АМКУ
- Рішення ВСУ по закупівлях
- Перехресні посилання з практикою ООПЗ

**Локальна спеціалізована LLM**
- Fine-tuning відкритої моделі на українській правовій базі:
  - 286K+ документів українського законодавства
  - 20K+ рішень ООПЗ
  - Рішення АМКУ і судів
  - Еталонні аналізи Magilarity
- Незалежність від зовнішніх API провайдерів
- Локальний деплой для клієнтів з вимогами конфіденційності

## 🌍 Фаза 4 — Вихід на ринок Європи

### Чому логіка Magilarity переноситься в ЄС

```
Єдина директива ЄС 2014/24 про публічні закупівлі
діє в усіх 27 країнах-членах ЄС

Спільна логіка для всіх:
├── Публічні закупівлі через єдині платформи
├── Антиконкурентні узгоджені дії (Art. 102 TFEU)
├── Органи оскарження (аналог ООПЗ)
├── Антимонопольні комітети (аналог АМКУ)
└── Відкриті дані закупівель

= Архітектура агентів переноситься
  змінюється тільки мова і локальне законодавство
```

### Пріоритетні ринки

**🇵🇱 Польща — перший ринок ЄС**
- Найбільший ринок публічних закупівель в ЦСЄ (~€50 млрд/рік)
- Схожі проблеми з корупцією в закупівлях
- Велика українська діаспора як перші клієнти
- Платформа: Platforma e-Zamówienia (аналог ProZorro)
- Орган оскарження: KIO (аналог ООПЗ)
- Строки локалізації: 1-2 місяці

**🇷🇴 Румунія**
- Активний ринок закупівель (~€15 млрд/рік)
- Значні проблеми з прозорістю
- Платформа: SEAP
- Висока зацікавленість антикорупційних організацій

**🇧🇬 Болгарія**
- Схожі системні проблеми із закупівлями
- Активна громадянська антикорупційна спільнота

**🇩🇪 Німеччина — довгострокова ціль**
- Найбільший ринок закупівель ЄС (~€500 млрд/рік)
- Платформа: DTVP, Vergabe24
- Вимагає повної локалізації та партнера

### Стратегія виходу в ЄС

```
КРОК 1 — Технічна підготовка:
├── Мовна локалізація (польська)
├── Завантаження бази польського законодавства
└── Підключення польської платформи закупівель API

КРОК 2 — Партнерство:
├── Юридична фірма-партнер в Польщі
├── Антикорупційна НКО як anchor client
└── Акселератор у ЦСЄ (Startup Wise Guys, EIT Digital)

КРОК 3 — Пілот:
├── 10-20 польських компаній-учасників закупівель
├── Верифікація якості висновків місцевим юристом
└── Адаптація промтів під польську правову практику

КРОК 4 — Масштабування:
└── Румунія → Болгарія → інші країни ЄС
```

## 💎 Стратегічне бачення

> *Magilarity — стати стандартом де-факто для аналізу публічних закупівель в Центральній та Східній Європі. Як YouControl став стандартом перевірки контрагентів в Україні — Magilarity стане стандартом для роботи з держзакупівлями.*

**Довгострокова мета:** Інфраструктурна платформа що обслуговує всіх учасників екосистеми державних закупівель — від малого бізнесу що вперше бере участь у тендері до антикорупційних органів що розслідують системні зловживання.
